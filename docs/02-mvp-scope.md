# MVP Scope

## MVP Goal

The goal of Quiet Bug MVP v0 is to create a small working version of the product that helps junior developers practise live coding interviews with structure, calm and confidence.

The MVP should focus on one clear problem:

Junior developers often freeze during live coding interviews because they do not know how to start, how to explain their thinking or how to recover when they get stuck.

## Main MVP Question

Can Quiet Bug help a junior developer feel less blocked during live coding practice?

## Included in MVP v0

The first version will include:

- Homepage
- Java practice page
- Exercise detail page
- Thinking checklist before coding
- Code textarea
- Panic recovery hints
- Oral explanation section
- Final solution section
- Interview feedback section
- Soft skills page
- Employer mindset content

## Not Included in MVP v0

The first version will not include:

- Login
- Payments
- Real code execution
- AI feedback
- Video recording
- Audio recording
- Multiple programming languages
- User progress tracking
- Admin dashboard
- Company-specific interview tracks
- Advanced algorithms

## Initial Programming Language

The first programming language will be Java.

Java was chosen because:

- it is widely used in backend development
- it is common in junior technical interviews
- it helps practise OOP concepts clearly
- it matches the first version of the product focus

Future versions may include:

- JavaScript
- SQL
- Python
- TypeScript
- C#

## MVP User

The MVP is designed for junior developers with 0 to 1 year of experience.

This includes:

- bootcamp graduates
- self-taught developers
- computer science graduates
- career changers
- junior backend candidates
- junior full-stack candidates

## Core Features

### 1. Java Practice

Users can choose from a list of beginner-friendly Java exercises based on common junior interview patterns.

### 2. Thinking Checklist

Before coding, users are guided to identify:

- input
- output
- whether a loop is needed
- whether a helper variable is needed
- edge cases
- explanation strategy

### 3. Panic Recovery

When users feel blocked, they can click an "I'm stuck" button.

The app will show small hints progressively instead of giving the full solution immediately.

### 4. Speak While Coding

Each exercise includes an example of how the candidate could explain their approach out loud during an interview.

### 5. Interview Feedback

Each exercise includes feedback about:

- technical reasoning
- communication
- edge cases
- what an employer may notice

### 6. Soft Skills

The MVP includes basic soft skills guidance for junior interviews, such as:

- how to answer when you do not know something
- how to explain a career change
- how to talk about a project
- how to receive feedback
- how to show learning mindset

## Success Criteria

The MVP is successful if a junior developer can:

- open the app
- understand the product purpose quickly
- choose a Java exercise
- follow the thinking checklist
- use panic recovery hints when stuck
- read an oral explanation example
- compare their answer with a solution
- feel more prepared for live coding interviews

## MVP Principle

Small, focused and useful.

It needs to prove that the product flow can help reduce live coding panic.
