# Content Plan

## Purpose

This document tracks the first content that will be created for Quiet Bug.

The MVP should start with a small set of Java exercises and soft skills topics for junior developers.

The goal is not to cover everything.

The goal is to create enough content to test the main product experience:

- coding practice
- thinking checklist
- panic recovery
- speak while coding
- interview feedback
- employer perspective

## Why These Exercises

These exercises were chosen because they match common junior interview patterns.

They help users practise:

- identifying input and output
- deciding if a loop is needed
- using helper variables
- writing small methods
- handling edge cases
- explaining logic clearly
- using basic OOP
- recovering when stuck

The goal is not advanced algorithms.

The goal is junior interview confidence.

## Content Rule

Every exercise should include:

- prompt
- example input
- expected output
- thinking checklist
- panic recovery hints
- oral explanation
- suggested solution
- interview feedback
- employer perspective
